# 在线推荐流程

## 1. 系统启动

文件：`backend/api/main.py`

服务按以下顺序初始化（`lifespan` 上下文管理器）：

| 顺序 | 服务 | 说明 |
|------|------|------|
| 1 | Redis（CacheService） | 连接 Redis，用于三级缓存 |
| 2 | EmbeddingService | 加载 `bge-base-en-v1.5` 模型 |
| 3 | VectorService | 连接 Milvus Lite，创建 `products` 和 `sid_mapping` 集合 |
| 4 | SIDService | 加载 `sid_mapping.json` 作为兜底数据 |
| 5 | RecommenderInference | 加载 Qwen2.5-3B + LoRA 模型，传入所有合法 SID |

所有服务挂载到 `app.state`，路由通过 `request.app.state.*` 访问。

**中间件**：
- `LoggingMiddleware` — 记录请求延迟和推荐指标
- `CORSMiddleware` — 允许前端跨域

**特殊端点**：
- `GET /health` — 健康检查
- `GET /metrics` — 汇总指标

## 2. 推荐接口

### 2.1 请求/响应

接口：`POST /api/v1/recommend`

**请求**：
```json
{
  "user_id": "demo_user",
  "history_asins": ["B001", "B002", "B003"],
  "top_k": 10
}
```

**响应**：
```json
{
  "user_id": "demo_user",
  "recommendations": [
    {
      "asin": "B007",
      "title": "Product Name",
      "description": "...",
      "category": "Tools > Drills",
      "brand": "BrandX",
      "price": 29.99,
      "rating": 4.5,
      "rating_count": 120
    }
  ],
  "total": 10,
  "model_version": "qwen2.5-3b-sft-grpo-v1"
}
```

### 2.2 完整处理流程

```
POST /recommend {user_id, history_asins, top_k}
       │
       ▼
┌─ Step 0: 缓存检查 ─────────────────────────────────┐
│  key = recommend:{user_id}:{md5(sorted_asins)}      │
│  命中 → 直接返回缓存结果                              │
└──────────────────────────────────────────────────────┘
       │ 未命中
       ▼
┌─ Step 1: ASIN → SID ───────────────────────────────┐
│  sid_svc.asins_to_sids(history_asins)                │
│  三级查询: Redis → Milvus → JSON                     │
│  记录 SID 命中率指标                                  │
│  无映射 → HTTP 400                                   │
└──────────────────────────────────────────────────────┘
       │ history_sids
       ▼
┌─ Step 2: 模型推理 ─────────────────────────────────┐
│  recommender.predict(history_sids)                   │
│  Qwen2.5-3B + LoRA, beam_search=20                  │
│  输出 top_k=10 个候选 SID                            │
│  记录推理耗时                                        │
└──────────────────────────────────────────────────────┘
       │ candidate_sids
       ▼
┌─ Step 3: SID → ASIN ───────────────────────────────┐
│  sid_svc.sids_to_asins(candidate_sids)               │
│  Milvus 查询 → JSON 兜底                             │
│  无有效 ASIN → HTTP 500                              │
└──────────────────────────────────────────────────────┘
       │ candidate_asins
       ▼
┌─ Step 4: 商品回查 ─────────────────────────────────┐
│  es_client.get_products_by_asins(asins[:top_k])      │
│  三级回查: Redis → Milvus → ES mget                  │
│  记录回查耗时                                        │
└──────────────────────────────────────────────────────┘
       │ products
       ▼
┌─ Step 5: 排序 ─────────────────────────────────────┐
│  按 candidate_asins 顺序排列（模型评分顺序）           │
└──────────────────────────────────────────────────────┘
       │
       ▼
┌─ Step 6: 写缓存 ───────────────────────────────────┐
│  Redis setex, TTL=300s (5分钟)                       │
└──────────────────────────────────────────────────────┘
       │
       ▼
  返回 RecommendResponse
```

## 3. 搜索接口

### 3.1 三模式搜索

接口：`GET /api/v1/products/search?q=...&mode=hybrid&size=20&category=...`

**keyword 模式**：
```
ES multi_match(query, fields=[title^3, description, brand^2])
  → 可选 category term 过滤
  → 返回 top size 条
```

**vector 模式**：
```
embedding_svc.encode_query(q) → 768 维向量
  → Milvus cosine 相似度搜索
  → 返回 top size 条
  → Milvus 不可用 → HTTP 503
```

**hybrid 模式（默认）**：
```
并行执行:
  ES keyword 搜索 → 取 size*2 候选
  Milvus 向量搜索 → 取 size*2 候选
      ↓
  向量无结果 → 降级为纯 keyword
      ↓
  RRF 融合 (k=60):
    score(doc) = Σ 1/(60+rank+1)
      ↓
  按分数降序，取 top size 条
```

### 3.2 RRF 融合示例

| 商品 | ES 排名 | Vector 排名 | RRF 分数 |
|------|---------|------------|---------|
| A | 0 | — | 1/61 = 0.0164 |
| B | 1 | 0 | 1/62 + 1/61 = 0.0325 |
| C | — | 1 | 1/62 = 0.0161 |

商品 B 同时出现在两个列表中，分数累加后排名最高。

## 4. 商品详情接口

### 4.1 单商品查询

接口：`GET /api/v1/products/{asin}`

调用 `es_client.get_products_by_asins([asin])`，三级回查后返回。未找到返回 404。

### 4.2 分页列表

接口：`GET /api/v1/products?page=1&page_size=20&category=...`

直接查询 Milvus 的 `products` 集合，支持类目筛选和分页。

## 6. 前端请求流

### 6.1 推荐请求

```
用户操作（点击/加入历史）
    ↓
behaviorStore.addRecord()
    ↓
historyAsins() 变化
    ↓
TanStack Query queryKey 变化
    ↓
自动触发 POST /api/v1/recommend
    ↓
渲染 ProductGrid
```

### 6.2 搜索请求

```
用户输入搜索词
    ↓
useProductSearch(query)
    ↓
GET /api/v1/products/search?q=...&mode=hybrid
    ↓
渲染搜索结果
```

### 6.3 API 客户端配置

文件：`frontend/src/api/client.ts`

- `baseURL: "/api/v1"`（开发环境 Vite 代理到后端）
- `timeout: 10000ms`
- 请求拦截器：注入 `X-User-Id` header（从 localStorage 读取）
- 响应拦截器：提取错误信息并重新抛出

## 7. 降级与容错

| 组件 | 故障模式 | 降级方案 |
|------|---------|---------|
| Redis | 连接失败 | 缓存禁用，所有方法静默返回空值 |
| Milvus | 连接失败 | 向量搜索禁用，hybrid 降级为 keyword |
| ES | 连接失败 | 全文搜索降级为 Milvus 内存子串匹配 |
| Embedding 模型 | 模型加载失败 | 生成确定性伪向量 |
| 推荐模型 | 模型加载失败 | 生成确定性伪随机 SID |
| SID 映射 | 三级查询均失败 | 丢弃该 ASIN；全部失败返回 400 |
| 模型无有效候选 | 所有 SID 反查失败 | 返回 HTTP 500 |

## 8. 配置参数

文件：`backend/config/settings.py`（Pydantic-Settings，读 `.env`）

| 分组 | 参数 | 默认值 | 说明 |
|------|------|--------|------|
| ES | `es_host` | `http://localhost:9200` | ES 地址 |
| ES | `es_index_name` | `industrial_products` | 索引名 |
| Redis | `redis_host` | `localhost` | Redis 地址 |
| Redis | `redis_recommend_ttl` | 300s | 推荐缓存 TTL |
| Redis | `redis_product_ttl` | 3600s | 商品缓存 TTL |
| Redis | `redis_sid_ttl` | 86400s | SID 缓存 TTL |
| Milvus | `milvus_uri` | `products.db` | Milvus Lite 本地文件 |
| 模型 | `model_base` | `Qwen/Qwen2.5-3B` | 基座模型 |
| 模型 | `beam_search_num_beams` | 20 | 束搜索宽度 |
| 模型 | `top_k` | 10 | 推荐候选数 |
| Embedding | `embedding_model` | `BAAI/bge-base-en-v1.5` | 向量模型 |
| Embedding | `embedding_dim` | 768 | 向量维度 |
