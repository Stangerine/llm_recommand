# 数据清洗与检索

## 1. 商品数据清洗

### 1.1 数据来源

原始数据来自 Amazon Reviews 的 `Industrial_and_Scientific` 类目，包含商品元数据（`meta_Industrial_and_Scientific.json`）和用户评论数据（`Industrial_and_Scientific.json`）。每条元数据为一行 JSON，包含 ASIN、标题、描述、类目、品牌、价格、评分等字段。

### 1.2 清洗流程

清洗脚本：`backend/data/scripts/preprocess_products.py`

逐条读取原始 JSONL，按以下规则处理各字段：

| 字段 | 提取逻辑 | 清洗规则 |
|------|---------|---------|
| `asin` | 优先取 `parent_asin`，回退取 `asin` | 跳过空值或 `"None"` |
| `title` | `item.get("title", "")` | `strip_html()` 去除 HTML 标签和实体，空标题跳过 |
| `description` | 列表用空格拼接，字符串直接使用 | `strip_html()` + 截断到 2000 字符，空描述跳过 |
| `category` | 兼容 `[["A","B"]]` 和 `["A","B"]` 两种格式 | 用 ` > ` 拼接成层级字符串 |
| `brand` | `item.get("brand", "")` | `strip_html()`，缺失默认空字符串 |
| `price` | `item.get("price")` | 去 `$` 和 `,`，转 float，校验 `0 < price < 100000`，缺失跳过 |
| `rating` | 优先 `average_rating`，回退 `rating` | 校验 `0-5` 范围，保留 1 位小数，缺失默认 `0.0` |
| `rating_count` | 优先 `rating_number`，回退 `rating_count` | 转 int，负值修正为 0 |

**必填字段过滤**：asin、title、description、price 任一缺失则整条记录丢弃。

**去重**：用 Pandas `drop_duplicates(subset=["asin"])` 按 ASIN 去重。

**输出**：`data/processed/products.jsonl`，每行一个 JSON 对象。

### 1.3 辅助函数

- `strip_html(text)`：先 `html.unescape()` 解码实体，再正则 `<[^>]+>` 去标签，最后合并空白
- `clean_price(price)`：支持 int/float/str 三种输入，去除 `$` 和 `,` 后转 float
- `clean_rating(rating)`：转 float 后校验 `0-5` 范围
- `clean_text(text, max_len)`：去 HTML 后截断

## 2. Elasticsearch 检索服务

### 2.1 索引创建

脚本：`backend/es_client/scripts/create_index.py`

连接 ES 后读取 mapping 文件，若索引已存在则跳过，否则创建。

### 2.2 索引 Mapping

配置文件：`backend/es_client/mappings/products_mapping.json`

**索引设置**：
- `number_of_shards: 1`（单机开发环境）
- `number_of_replicas: 0`
- 自定义分词器 `product_analyzer`：standard 分词器 + 英文停用词过滤

**字段映射**：

| 字段 | ES 类型 | 分词器 | 说明 |
|------|--------|--------|------|
| `asin` | keyword | — | 精确匹配，用作文档 `_id` |
| `title` | text | `product_analyzer` | 全文检索，带 `.raw` keyword 子字段 |
| `description` | text | `product_analyzer` | 全文检索 |
| `category` | keyword | — | 精确匹配，可筛选 |
| `brand` | keyword | — | 精确匹配 |
| `price` | float | — | `null_value: -1.0` |
| `rating` | float | — | `null_value: 0.0` |
| `rating_count` | integer | — | `null_value: 0` |
| `sid` | keyword | — | 语义 ID |

### 2.3 批量写入

脚本：`backend/es_client/scripts/bulk_index_products.py`

1. 加载 `sid_mapping.json` 中的 `asin2sid` 映射
2. 逐行流式读取 `products.jsonl`（内存友好）
3. 每条文档合并产品字段 + `sid` 字段，`_id` 设为 ASIN
4. 使用 `elasticsearch.helpers.bulk()` 批量写入，`chunk_size=500`

### 2.4 搜索模式

ES 客户端：`backend/es_client/client.py`

**全文检索**（`search_products` 方法）：
- 使用 `multi_match` 查询，字段权重：`title^3`、`description`、`brand^2`
- 可选 `category` term 过滤
- 包装在 `bool.must` 中

**降级方案**：ES 不可用时，从 Milvus 取 500 条商品做内存子串匹配。

## 3. Milvus 向量检索服务

### 3.1 商品集合（`products`）

向量服务：`backend/services/vector_service.py`

| 字段 | 类型 | 约束 |
|------|------|------|
| `asin` | VARCHAR(64) | PRIMARY KEY |
| `title` | VARCHAR(2048) | |
| `description` | VARCHAR(4096) | |
| `category` | VARCHAR(1024) | |
| `brand` | VARCHAR(256) | |
| `price` | FLOAT | |
| `rating` | FLOAT | |
| `rating_count` | INT64 | |
| `embedding` | FLOAT_VECTOR(768) | |

索引类型：**FLAT**（暴力搜索），距离度量：**COSINE**。适合内存可容纳的数据集。

**写入逻辑**（`build_embeddings.py`）：
1. 拼接文本 `"{title} {description[:200]}"` 作为 embedding 输入
2. 使用 `BAAI/bge-base-en-v1.5` 生成 768 维 L2 归一化向量
3. 批量插入（500 条/批），写入前删除旧库保证幂等

### 3.2 SID 映射集合（`sid_mapping`）

| 字段 | 类型 | 约束 |
|------|------|------|
| `asin` | VARCHAR(64) | PRIMARY KEY |
| `sid` | VARCHAR(32) | |
| `dummy_vector` | FLOAT_VECTOR(1) | Milvus Lite 要求至少一个向量字段 |

该集合用于 ASIN ↔ SID 双向查询，不依赖向量相似度。

### 3.3 Embedding 服务

服务：`backend/services/embedding_service.py`

- 模型：`BAAI/bge-base-en-v1.5`（768 维）
- 批量编码：`batch_size=128`，`normalize_embeddings=True`
- 降级模式：模型不可用时，用 `hash(text)` 生成确定性伪向量

## 4. Redis 缓存服务

### 4.1 三级缓存策略

服务：`backend/services/cache_service.py`

| 缓存域 | Key 格式 | TTL | 用途 |
|--------|---------|-----|------|
| 推荐结果 | `recommend:{user_id}:{history_hash}` | 5 分钟 | 相同用户相同历史不重复推理 |
| 商品详情 | `product:{asin}` | 1 小时 | 推荐/搜索结果回查加速 |
| SID 映射 | `sid:{asin}` | 24 小时 | ASIN↔SID 转换，变化频率低 |

### 4.2 批量操作

使用 Redis pipeline 减少网络 RTT：
- `get_products_batch(asins)` — `mget` 批量读取
- `set_products_batch(products)` — pipeline 批量写入
- `get_sids_batch(asins)` / `set_sids_batch(mappings)` — 同理

### 4.3 降级策略

Redis 不可用时所有方法静默返回 `None`/`False`/`{}`，系统继续运行（无缓存模式）。

## 5. 混合检索与 RRF 融合

### 5.1 三模式搜索

接口：`GET /api/v1/products/search`

| 模式 | 实现 | 说明 |
|------|------|------|
| `keyword` | ES `multi_match`（title×3, brand×2） | 纯关键词匹配 |
| `vector` | Milvus 余弦相似度（bge-base-en-v1.5） | 纯语义检索 |
| `hybrid`（默认） | 两者并行 + RRF 融合 | 关键词 + 语义，互补召回 |

### 5.2 RRF 融合算法

```python
score(doc) = Σ 1 / (k + rank + 1)
```

- `k = 60`（平滑常数，防止头部结果过度主导）
- 分别从 ES 和 Milvus 取 `size * 2` 候选
- 同一文档出现在两个列表中时分数累加
- 按分数降序排列，取 top `size` 返回

### 5.3 三级产品回查

`ESClient.get_products_by_asins()` 实现三级降级：

1. **Redis 缓存** — `mget` 批量查，命中直接返回
2. **Milvus** — `asin in [...]` 过滤查询，结果回填 Redis
3. **ES mget** — 批量 ID 查询，兜底方案
4. **占位符** — 仍缺失的 ASIN 返回带标题的占位对象

结果保持原始 ASIN 顺序，确保推荐排序不受存储层影响。
