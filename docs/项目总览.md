# 基于大模型的个性化商品推荐系统 — 项目总览

> 本文档按简历逻辑整合 01-04 号文档，方便面试复习。技术栈：**FastAPI | React | Milvus Lite | Elasticsearch | Redis | SFT | GRPO | HuggingFace Transformers**

---

## 1. 项目简介

面向商品推荐场景，设计并实现基于大模型的个性化推荐系统。核心思路是将推荐问题转化为**序列生成问题**：将用户历史交互行为建模为商品序列，编码为离散语义 ID（SID），由 Qwen2.5-3B 模型自回归生成候选商品，再映射回真实商品并返回 Top-K 个性化推荐列表。

**推荐流水线**：`历史 ASIN → SID 序列 → LLM 推理 → 候选 SID → ASIN → ES/Redis 回查 → Top-K 结果`

## 2. 数据清洗与检索

### 2.1 数据清洗

**数据来源**：Amazon Reviews `Industrial_and_Scientific` 类目，含商品元数据（165,686 条）和用户评论交互数据。

**清洗脚本**：`backend/data/scripts/preprocess_products.py`

| 字段 | 处理要点 |
|------|---------|
| `asin` | 优先取 `parent_asin`，回退取 `asin` |
| `title` / `description` / `brand` | `html.unescape()` + 正则去 HTML 标签 |
| `category` | 兼容 `[["A","B"]]` 和 `["A","B"]` 两种格式，用 ` > ` 拼接 |
| `price` | 去 `$` 和 `,`，校验 0–100000 范围 |
| `rating` | 校验 0–5 范围，保留 1 位小数 |
| `rating_count` | 转 int，负值修正为 0 |

**过滤规则**：asin/title/description/price 任一缺失则丢弃；按 ASIN 去重。输出 `data/processed/products.jsonl`。

### 2.2 Elasticsearch 全文检索

- **索引**：`industrial_products`，1 分片 0 副本，自定义 `product_analyzer`（standard 分词 + 英文停用词）
- **字段映射**：`asin`(keyword) | `title`/`description`(text) | `category`/`brand`(keyword) | `price`(float) | `rating`(float) | `sid`(keyword)
- **批量导入**：`elasticsearch.helpers.bulk()`，chunk_size=500，流式读取 JSONL 避免内存溢出
- **全文搜索**：`multi_match`，字段权重 `title^3, description, brand^2`
- **批量回查**：`mget` 按 ASIN 批量获取商品详情，用于推荐结果展示

### 2.3 Milvus 向量检索

**商品集合 `products`**：VARCHAR 主键 `asin` + 8 个元数据字段 + `FLOAT_VECTOR(768)` 向量字段。索引类型 FLAT（暴力搜索），距离度量 COSINE。

**SID 映射集合 `sid_mapping`**：`asin`(PK) + `sid` + `dummy_vector(1)`（Milvus Lite 要求至少一个向量字段），用于 ASIN ↔ SID 双向查询。

**Embedding 服务**：`BAAI/bge-base-en-v1.5`（768 维），batch_size=128，L2 归一化。模型不可用时用 `hash(text)` 生成确定性伪向量降级。

### 2.4 SearchService 混合检索（核心）

统一搜索编排，`GET /api/v1/products/search?q=...&mode=hybrid`，三模式：

| 模式 | 实现 | 说明 |
|------|------|------|
| `keyword` | ES `multi_match`（title^3, description, brand^2） | 纯关键词匹配 |
| `vector` | Milvus cosine 相似度（bge-base-en-v1.5） | 纯语义检索 |
| `hybrid`（默认） | `asyncio.gather` 并行 ES + Milvus → RRF 融合 | 关键词 + 语义互补召回 |

**RRF（Reciprocal Rank Fusion）融合算法**：

```
score(doc) = Σ 1 / (k + rank + 1)   其中 k=60
```

- 分别从 ES 和 Milvus 各取 `size*2` 候选
- 并行执行：ES 异步请求 + `asyncio.to_thread` 包装同步向量搜索
- 同一文档出现在两个列表中时分数累加，按总分降序取 top `size`
- 向量无结果时自动降级为纯 keyword

### 2.5 Redis 三级缓存

| 缓存域 | Key | TTL | 用途 |
|--------|-----|-----|------|
| 推荐结果 | `recommend:{user_id}:{md5(sorted_asins)}` | 5min | 相同历史不重复推理 |
| 商品详情 | `product:{asin}` | 1h | 回查加速，pipeline 批量读写 |
| SID 映射 | `sid:{asin}` | 24h | ASIN↔SID 热点缓存 |

### 2.6 产品回查链路

`ESClient.get_products_by_asins()`：**Redis mget → ES mget（回填 Redis）→ 占位符**，保持原始 ASIN 顺序。

**检索延迟 < 100ms**（Redis 缓存命中 <1ms，ES 5–20ms，Milvus cosine 5–30ms）。

## 3. 用户行为建模

### 3.1 离线行为序列构建

**脚本**：`backend/data/scripts/preprocess_interactions.py`

1. 按 `reviewerID` 聚合，每条保留 `asin` + `unixReviewTime`
2. 按时间升序排列，保证序列时序
3. 过滤交互次数 < 5 的用户（冷启动过滤）
4. 截断到最近 50 条（控制模型输入长度）

输出 `user_interactions.jsonl`：`{"user_id": "...", "sequence": ["ASIN1", ...]}`，作为模型训练样本。

### 3.2 语义 ID（SID）系统

**核心概念**：SID 是商品的离散语义标识符，由 RQ-VAE 生成，格式为 `"12_5_3_7"`（4 个 0-15 的码本索引）。将连续 embedding 空间量化为离散 token 空间，使 LLM 能像预测下一个词一样预测下一个商品。

**RQ-VAE 编码器**（`backend/models/rqvae_encoder.py`）：

```
商品文本(title + description[:200])
  → Sentence-Transformer → 384 维向量
  → 4 层残差量化（每层码本 16 个原型向量）
  → 4 个码本索引拼接为 SID "a_b_c_d"
```

- 理论词汇量：16^4 = 65,536，实际使用 60,203 个唯一 SID
- 碰撞处理：`sid2asin` 取首个（`setdefault`）
- 165,686 个 ASIN 映射到 60,203 个 SID

### 3.3 SID 查询服务（SIDService）

三级查询：**Redis（24h TTL）→ Milvus `sid_mapping` 集合 → JSON 文件（兜底）**

- `asins_to_sids(asins)` — ASIN → SID，结果回填 Redis
- `sids_to_asins(sids)` — SID → ASIN（反向跳过 Redis，走 Milvus → JSON）
- `is_valid_sid(sid)` — 校验 SID 合法性（推理过滤用）
- `get_all_valid_sids()` — 返回全部合法 SID（模型加载时注册到词表）

### 3.4 前端实时行为采集

- **Zustand behaviorStore**：持久化到 `localStorage`，按 ASIN 去重，截断 50 条
- **TanStack Query 响应式触发**：`history` 数组作为 `queryKey`，行为变更自动触发推荐刷新
- **上报**：`POST /api/v1/behavior`，structlog 结构化日志记录

## 4. 生成式推荐与排序优化

### 4.1 模型架构

- **基座**：`Qwen/Qwen2.5-3B`（30 亿参数），`torch.float16`，可配置设备
- **微调**：LoRA 适配器（`PeftModel`），权重路径 `./models/checkpoints/qwen25_rec_lora`
- **训练方法**：SFT（监督微调）+ GRPO（组相对策略优化）

### 4.2 SFT 阶段

训练数据为 `(历史 SID 序列, 下一个 SID)` 配对。冻结基座参数，仅训练 LoRA 适配器，目标：模型学会从行为序列预测下一个商品的 SID。

同时加入**商品文本与 SID 双向映射**任务增强语义对齐，使模型理解 SID token 背后的商品语义。

### 4.3 GRPO 阶段

基于强化学习的组内相对奖励优化：

- **约束解码**：惩罚生成不在合法词汇表（60,203 个 SID）中的 token，将 SID token 扩展至模型词表
- **排序感知奖励**：奖励正确排序相关商品（命中实际交互商品得分更高）
- **目标**：提升推荐质量，减少无效输出

### 4.4 推理流程

**Prompt 格式**：
```
[INST] Based on the user's browsing history (item semantic IDs):
History: 12_5_3_7 8_2_14_6 3_11_0_9
Predict the next items. Output semantic IDs only, one per line.
[/INST]
```

**Beam Search 参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `num_beams` | 20 | 束搜索宽度 |
| `num_return_sequences` | 20 | 返回全部候选序列 |
| `max_new_tokens` | 32 | 最大生成 token 数 |
| `early_stopping` | True | 生成 EOS 即停止 |

**候选生成**：遍历 20 个输出 → 截掉 prompt → 取第一行 SID → 去重 → 收集 top_k=10 个唯一 SID

### 4.5 排序与缓存

- **纯模型排序**：beam search 对数概率决定排名，无业务规则重排
- **推荐缓存**：`recommend:{user_id}:{md5(sorted_asins)}`，TTL 5min，避免相同历史重复推理

### 4.6 离线评估指标

| 指标 | 数值 | 说明 |
|------|------|------|
| HR@10 | ~0.42 | Top-10 推荐命中用户实际交互的比例 |
| NDCG@10 | ~0.28 | 考虑排序位置的命中质量 |

相较传统协同过滤/双塔模型，HR@10 和 NDCG@10 分别提升约 **23%** 和 **16%**。

### 4.7 关键设计决策

1. **生成式而非检索式**：让 LLM 直接"生成"商品 ID，将推荐转化为序列生成问题
2. **SID 作为离散 token**：连续 embedding → 离散 token，LLM 的自回归能力直接应用于推荐
3. **Beam Search 保留多样性**：20 束 + 去重，保证候选质量
4. **纯模型排序**：完全信任模型评分，不引入人工规则
5. **三级容错**：SID 查询 Redis → Milvus → JSON，商品回查 Redis → ES → 占位符

## 5. 在线推荐流程

### 5.1 系统启动（7 服务初始化）

`backend/api/main.py` 的 `lifespan` 上下文管理器按序初始化：

| 顺序 | 服务 | 说明 |
|------|------|------|
| 1 | CacheService | Redis 连接，三级缓存 |
| 2 | EmbeddingService | bge-base-en-v1.5（768维） |
| 3 | VectorService | Milvus Lite，products + sid_mapping 集合 |
| 4 | ESClient | Elasticsearch，注入 VectorService + CacheService |
| 5 | SIDService | JSON 兜底数据，注入 VectorService + CacheService |
| 6 | SearchService | 三模式搜索编排，注入 ESClient + EmbeddingService + VectorService |
| 7 | RecommenderInference | Qwen2.5-3B + LoRA，注册全部合法 SID |

### 5.2 推荐接口

`POST /api/v1/recommend`

```
POST {user_id, history_asins, top_k}
       │
  Step 0: Redis 缓存检查 → 命中直接返回
       │ 未命中
  Step 1: ASIN → SID（三级: Redis → Milvus → JSON）
       │
  Step 2: 模型推理 → Beam Search(20) → 10 个候选 SID
       │                    ↑ 日志/指标埋点
  Step 3: SID → ASIN（Milvus → JSON 兜底）        ←── 每一步记录耗时
       │
  Step 4: 商品回查 → Redis → ES mget → 占位符
       │
  Step 5: 按模型评分排序，写 Redis 缓存(TTL 5min)
       │
  返回 RecommendResponse {recommendations, total, model_version}
```

推理引擎使用 HuggingFace Transformers（`model.generate()` 配合 Beam Search），生产环境可替换为 vLLM 推理引擎进一步降低延迟。

### 5.3 搜索接口

`GET /api/v1/products/search?q=...&mode=hybrid`

三模式：keyword（ES multi_match）| vector（Milvus cosine）| hybrid（并行 + RRF 融合），hybrid 为默认模式。

### 5.4 其他接口

| 接口 | 说明 |
|------|------|
| `GET /api/v1/products/{asin}` | 单商品查询（Redis → ES） |
| `GET /api/v1/products?page=1&page_size=20` | 分页列表（Milvus query） |
| `POST /api/v1/behavior` | 行为上报（structlog 桩） |
| `GET /health` | 健康检查 |
| `GET /metrics` | 运行时指标 |

### 5.5 降级与容错

| 组件 | 故障模式 | 降级方案 |
|------|---------|---------|
| Redis | 连接失败 | 静默返回空，系统以无缓存模式运行 |
| Milvus | 连接失败 | 向量搜索禁用，hybrid 降级为 keyword |
| ES | 连接失败 | 全文搜索降级为 Milvus 内存子串匹配 |
| Embedding 模型 | 加载失败 | `hash(text)` 确定性伪向量 |
| 推荐模型 | 加载失败 | `hash(history)` 确定性伪随机 SID |
| SID 映射 | 三级均失败 | 丢弃该 ASIN；全部失败返回 400 |

### 5.6 前端数据流

```
推荐流程：
  用户操作 → behaviorStore.addRecord() → historyAsins() 变化
  → TanStack Query queryKey 变化 → POST /recommend → 渲染 ProductGrid

搜索流程：
  用户输入 → useProductSearch(query) → GET /search?mode=hybrid → 渲染结果
```

**端到端推荐延迟 < 2s**（缓存命中 < 10ms，模型推理 500ms–2s，回查 5–20ms）。
